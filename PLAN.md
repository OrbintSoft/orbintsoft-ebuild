## Decisions taken

- Plan stored in `CLAUDE.md` (rules) + `PLAN.md` (roadmap).
- Start from **Phase 0**.
- Repository to be renamed `local` â†’ `orbintsoft`.
- License: **GPL-3** for the whole repo.
- Linters: **pkgcheck** + **pkgdev** (canonical Gentoo QA) + **shellcheck** for scripts.
- CI: GitHub Actions, **test only packages changed in the PR**, in a `gentoo/stage3` container.
- Publishing: as a **git overlay on GitHub, no server needed**.
- Version-bump automation: **two layers.** (1) **Dependabot/Renovate** for GitHub
  Actions pins only â€” they cannot parse ebuilds. In this repo the only pins they can
  follow are the action versions (`actions/checkout`, `actions/setup-go`); the Go
  tools and `gentoo/stage3` are `@latest` (unpinned). The two bots overlap fully
  here, so the plan is **compare-then-keep-one**, not both forever. (2) An **ebuild
  bump bot**: evaluate **[Tatsh/livecheck](https://github.com/Tatsh/livecheck)**
  (nvchecker-inspired; `--auto-update --git`, GitHub/PyPI/Repology/â€¦, pkgdev
  integration, handles live `-9999` via commit SHA; v0.2.4 2026-05-31) before
  building a custom nvchecker-based bot.
- **Live `-9999` ebuilds stay as-is for now** (all packages are live, that's fine).
  Liveâ†’versioned conversion is future, per-package, decided dep by dep â€” only after
  CI + bump automation exist.
- **md5-cache**: not hand-committed â€” gitignored and regenerated by `egencache`/CI.
- **EAPI 9** (released 2025-12-14, supported by the installed Portage) is the target
  for new/bumped ebuilds; existing ebuilds migrate in Phase 1 (1.10). Future EAPIs
  adopted likewise when released.
- **Commit messages**: Conventional Commits as a *should*, not a *must* (see CLAUDE.md).
- **Copyright headers**: three tiers â€” original files â†’ `Stefano Balzarotti` + GPL-3;
  files reworked from a Gentoo(-Authors) ebuild â†’ dual `Gentoo Authors` + `Stefano
  Balzarotti` + GPL-2 (pamtester, tvision, bt-keys-sync, fsearch, claude-desktop,
  nerd-fonts). `pkgcheck scan` does **not** flag GPL-3 or non-Gentoo-Authors
  headers (verified 2026-06-12), so the open question on this is resolved.
